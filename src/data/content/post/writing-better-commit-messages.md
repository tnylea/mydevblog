### Good commit message format:

```
type(scope): description
```

Example:

```
feat(auth): add OAuth login support
```

### Types:
- feat: new feature
- fix: bug fix
- docs: documentation
- refactor: code change (non-feature/bug)

Clear messages make history easier to navigate!
